DMgtAsia · Session-III

The Simple Features standard

  • sf implements Simple Features, the ISO standard for representing geometry.
  • An sf object is a data frame with a special geometry column.
  • So spatial data is just tidy data with a place attached: one row per feature, columns of attributes, plus geometry.

Live R cells on this page run in your browser via webR. The first cell takes a few seconds while sf loads; after that they are quick.

Geometry types

  • POINT a single location (a clinic, a case).
  • LINESTRING a path (a road, a river).
  • POLYGON an area (a district, a PHC catchment).
  • Each has a MULTI form (MULTIPOLYGON for a state made of many parts).

Coordinate reference systems

  • A CRS tells R what the coordinates mean.
  • Geodetic (EPSG:4326, WGS 84): longitude/latitude in degrees. Good for storage and web maps.
  • Projected (a UTM zone such as EPSG:32643): metres on a flat plane. Needed for distances and areas.

Reading spatial files

In RStudio you read a shapefile or GeoJSON from disk:

library(sf)
library(here)
india <- st_read(here("spatial_files", "India_State_Boundary.shp"))
st_crs(india)

You do this in Exercise 1. We keep file reading in RStudio because the data files are large for a browser session.

Recap

  • sf is tidy data with a geometry column.
  • Geometry types: point, line, polygon, and their multi forms.
  • st_as_sf builds geometry; st_crs reads it; st_transform reprojects.
  • geom_sf maps it.
  • Heavier tools (raster, geocoding, clustering) run in RStudio and in Session-IV.
